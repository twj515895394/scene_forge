# SceneForge 仓库级运行规则

本文件定义 SceneForge 的仓库级运行期约束，供执行阶段的 agent 统一遵循。

目标只有两个：

- 统一主链 skill 里反复出现的通用规则
- 在不删除各 skill 关键提醒的前提下，减少重复解释和上下文漂移

冲突优先级：

- 用户当前明确指令
- 当前任务命中的 skill
- 本文件

## 1. 运行期上下文读取边界

默认只允许读取当前项目执行所必需的材料。

运行时默认禁止主动读取：

- `docs/`
- `.handoff/`
- `projects/` 下当前项目之外的其他项目目录
- 历史项目输出
- 与当前阶段无关的目录或资产

只有在以下情况才允许越界读取：

- 用户明确要求参考某个历史项目
- 当前任务就是做迁移、复刻、对比或复用分析
- 当前 skill 明确要求读取某个额外运行期资产文件

默认原则：

- 不主动把历史项目产物当参考库
- 不为了“找灵感”扫描其他项目
- 不把 `docs/` 或 `.handoff/` 当作执行期输入

## 2. 紧凑上下文预算

执行期优先采用 `compact` 上下文预算。

默认优先读取：

- 当前项目 `PROJECT_BOARD.md`
- 当前命中的 skill
- 当前 skill 的 `references/output-contract.md`
- 当前阶段在黑板中明确索引的文件
- 当前项目 `details/`、`outputs/`、`inputs/` 中与本阶段直接相关的文件

默认不做的事：

- 不全仓库扫描
- 不默认读取长文件全文
- 不默认读取历史 source 解析全文
- 不默认读取与本阶段无关的资产库

如果确实需要读取长文件或扩展资产，应该：

- 只读取相关章节或目标片段
- 明确说明读取原因

## 3. 轻黑板纪律

`PROJECT_BOARD.md` 是轻黑板，不是长正文仓库。

统一要求：

- 黑板只保留状态、摘要、索引、版本、路径、路由信息
- 长正文一律外部落盘
- 不把完整剧本、完整分镜、完整提示词、完整声音方案直接塞进黑板

默认回写格式：

- `board_updates`
- `files_created`
- `files_updated`
- `next_action`

完整内容应落到实际文件，再通过 `stage_index.*.files` 和摘要回写黑板。

## 4. 预览先行，确认后正式落盘

凡是会生成正式阶段产物的主链 skill，默认先出预览，再等用户确认。

统一要求：

- 先输出阶段预览或方案预览
- 用户明确确认后，才允许正式落盘
- 用户纠错、补充偏好或讨论方向，不等于授权正式写文件或推进状态

如果阶段本身要求显式确认，则不得跳过。

## 5. 产物诚实性规则

SceneForge 默认输出的是结构化资料、提示词、说明文档和阶段补丁，不直接声称已经完成外部生成结果。

统一要求：

- `scene-design-builder` 输出设计说明和图片提示词，不声称已生成图片
- `scene-storyboard-director` 输出分镜资料和故事板 prompt，不声称已生成故事板图
- `scene-video-prompt-builder` 输出视频提示词，不声称已生成视频
- `scene-video-intake` 输出解析资料和 handoff，不声称已生成图片、视频或音频

如果外部平台生成是用户手动完成的，必须明确说明这一点。

## 6. 与各 skill 的关系

本文件只负责仓库级通用规则，不替代各 skill 的阶段细节。

执行约定：

- 各 skill 内已经存在的关键提醒先保留
- 本文件负责统一口径
- skill 内规则负责在长上下文中再次提醒

如果未来要继续瘦身，优先压缩重复解释，不删除关键边界提醒。
