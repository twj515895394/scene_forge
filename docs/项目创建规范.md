# 项目创建规范

## 项目结构

每个具体片段项目位于 `projects/项目名/` 下。

```text
projects/
  2026-06-西游记-三打白骨精/
    PROJECT_BOARD.md
    details/
    outputs/
```

## PROJECT_BOARD.md

`PROJECT_BOARD.md` 是每个项目的唯一状态源。

它保存：

- 顶层索引字段（项目状态、下一阶段、生命周期、关键全局索引）
- 各阶段结构化 YAML 补丁块
- 关键决策
- 资产引用
- 风险摘要
- 产物版本引用
- 待办事项

顶层 YAML 当前统一字段为：

```yaml
project_name:
project_status:
next_stage:
lifecycle_flag:
blocker_note:
score:
production_level:
reference_type:
adaptation_level:
performance_style:
segment_duration_seconds:
created_at:
updated_at:
```

每个阶段分区统一使用结构化 YAML 补丁块，补丁壳如下：

```yaml
patch_type:
version:
status:
updated_at:
summary:
data:
```

显示规则统一为：

- 对话与黑板概览：中文优先，必要时带英文参数值
- `summary`：使用中文描述，建议在关键状态后附英文参数值
- 结构化字段：继续使用英文参数值，作为唯一机器可读口径

## details/

`details/` 是可选长内容区，不保存 Agent 思考过程，不保存废稿。

只保存正式但过长、不适合放在黑板里的内容：

- 完整剧本
- 完整分镜表
- 复杂参考对象分析
- 复杂风险说明
- 角色/场景锁定卡

## outputs/

`outputs/` 只保存可直接拿去使用的最终产物：

- 角色设定图 prompt
- 场景设定图 prompt
- 故事板提示词
- 视频分段提示词
- 发布文案
- 最终图片
- 最终视频
- 字幕
- 配音文案

视频提示词当前不按模型分类，保持通用目录。

默认视频分段时长为 `10` 秒，对应顶层字段 `segment_duration_seconds`。若分镜阶段判断 10 秒承载不下关键表演，可在用户确认后改为 `15`。
