# 项目创建规范

## 项目结构

每个具体片段项目位于 `projects/项目名/` 下。

```text
projects/
  2026-06-西游记-三打白骨精/
    PROJECT_BOARD.md
    details/
    outputs/
```

## PROJECT_BOARD.md

`PROJECT_BOARD.md` 是每个项目的唯一状态源，也是具体项目的轻量记忆层。

它保存：

- 顶层索引字段（项目状态、下一阶段、生命周期、关键全局索引）
- 上下文读取策略与 Token 预算
- 用户确认状态
- 时长与视频分段策略
- Hero Moment / 看点镜头
- Bridge Shot / 桥接分镜
- Blocking Map / 空间调度图
- Faction Layout / 阵营布局
- Prop State Machines / 道具状态机
- 各阶段结构化 YAML 补丁块
- 关键决策
- 资产引用
- 风险摘要
- 产物版本引用
- 待办事项

顶层 YAML 当前统一字段为：

```yaml
project_name:
project_status:
next_stage:
lifecycle_flag:
blocker_note:
score:
production_level:
reference_type:
adaptation_level:
performance_style:
target_total_duration_seconds:
segment_duration_seconds:
context_policy:
user_confirmations:
segment_strategy:
hero_moments:
bridge_shots:
prop_state_machines:
blocking_map:
faction_layout:
created_at:
updated_at:
```

建议初始化默认值：

```yaml
context_policy:
  mode: minimal
  allow_docs_scan: false
  active_protocol_docs: []
  forbidden_by_default:
    - docs/
    - .handoff/
    - 会话记录_*.md
    - 历史项目输出
  token_budget:
    default_stage_budget: compact
    require_reason_for_extra_reads: true

user_confirmations:
  duration_strategy_confirmed: false
  script_plan_confirmed: false
  design_direction_confirmed: false
  storyboard_plan_confirmed: false
  video_prompt_plan_confirmed: false

segment_strategy:
  target_total_duration_seconds:
  segment_duration_seconds: 10
  segment_count:
  segmentation_mode: fixed_10s
  confirmation_note:

hero_moments: []
bridge_shots: []
prop_state_machines: []
blocking_map:
  spatial_axis:
  zones: []
  continuity_rule:
faction_layout:
  factions: []
```

说明：

- `context_policy` 是执行阶段的上下文读取边界，默认不扫描 `docs/`、`.handoff/` 和历史项目输出。
- `user_confirmations` 记录用户是否确认关键创作决策。用户纠错或补充偏好不等于授权落盘。
- `segment_strategy` 记录整片目标时长、单段视频生成时长和分段模式。
- `hero_moments` 记录当前项目自己的看点镜头，不得复用某个样例项目的固定看点。
- `bridge_shots` 记录当前项目 Segment 之间的桥接分镜。
- `prop_state_machines` 记录当前项目核心道具的状态变化。
- `blocking_map` 与 `faction_layout` 记录当前项目角色站位、允许区域、禁止区域和阵营关系。

每个阶段分区统一使用结构化 YAML 补丁块，补丁壳如下：

```yaml
patch_type:
version:
status:
updated_at:
summary:
data:
```

显示规则统一为：

- 对话与黑板概览：中文优先，必要时带英文参数值
- `summary`：使用中文描述，建议在关键状态后附英文参数值
- 结构化字段：继续使用英文参数值，作为唯一机器可读口径

## 阶段确认闸门

以下阶段默认必须先给方案预览，用户确认后才允许落盘正式文件或推进阶段：

- `scene-script-adapter`：确认总时长、分段策略、剧本方案和 Story Beat 方向。
- `scene-design-builder`：确认角色方向、场景道具清单、统一视觉语言和参考强度。
- `scene-storyboard-director`：确认分镜结构、Hero Shot、Bridge Shot、Segment Plan 和 Blocking Map。
- `scene-video-prompt-builder`：确认分段提示词结构、参考图使用方案和连续性策略。

## details/

`details/` 是可选长内容区，不保存 Agent 思考过程，不保存废稿。

只保存正式但过长、不适合放在黑板里的内容：

- 完整剧本
- 完整分镜表
- 复杂参考对象分析
- 复杂风险说明
- 角色/场景/道具锁定卡
- 表演表
- 声音方案

## outputs/

`outputs/` 只保存可直接拿去使用的提示词、制作说明和发布素材文案。

当前 SceneForge 不负责生成图片、视频或音频。图片、视频和音频需要用户基于 prompt 在外部平台手动生成。

允许保存：

- 角色设定图 prompt
- 场景设定图 prompt
- 关键道具设定图 prompt
- 全场景资产总参考图 prompt
- 故事板图 prompt
- 视频分段提示词
- 一致性规则说明
- 声音制作说明 / 音乐 prompt / 拟音 prompt
- 发布文案
- 字幕文案
- 配音文案

不应默认保存或声称已生成：

- 最终图片
- 最终视频
- 最终音频
- 已嵌入文档的生成图

视频提示词当前不按模型分类，保持通用目录。

默认视频分段时长为 `10` 秒，对应顶层字段 `segment_duration_seconds`。若分镜阶段判断 10 秒承载不下关键表演，可在用户确认后改为 `15` 或混合分段。
