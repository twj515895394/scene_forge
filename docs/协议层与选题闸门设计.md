# 协议层与主链路设计基线

## 文档目的

本文档用于冻结 SceneForge 第一版黑板协议，以及主链路前半段核心 Skill 的设计基线。

目标只有两个：

1. 让总控 Skill `scene-forge` 能稳定读取项目状态并路由下一步。
2. 让所有子 Skill 后续按同一套补丁协议回写 `PROJECT_BOARD.md`。

## 适用范围

- `projects/*/PROJECT_BOARD.md` 顶层 YAML 索引字段
- 各阶段分区补丁结构
- `scene-topic-gate`
- `scene-reference-decider`
- 显示规范与主链路状态推进口径

本文档不覆盖以下内容：

- 资产卡模板字段设计
- 各后续 Skill 的完整细粒度 schema 细节

## 设计原则

1. 顶层 YAML 只保留跨阶段共用的薄索引字段，避免重复写入。
2. 各阶段细节只落在对应分区补丁里，避免顶层和分区双写漂移。
3. 补丁格式优先机器可解析，同时保留最小的人类可读摘要。
4. 主状态机只表达主流程阶段，异常态单独建模。

## `PROJECT_BOARD.md` 顶层协议

顶层 YAML 使用薄索引模式，统一字段如下：

```yaml
---
project_name:
project_status:
next_stage:
lifecycle_flag:
blocker_note:
score:
production_level:
reference_type:
adaptation_level:
performance_style:
created_at:
updated_at:
---
```

### 字段说明

- `project_name`：项目名。
- `project_status`：当前主流程阶段。
- `next_stage`：下一建议阶段，供总控 Skill 路由使用。
- `lifecycle_flag`：项目生命周期标记。
- `blocker_note`：阻塞原因摘要；无阻塞时留空。
- `score`：选题总分，全局索引值。
- `production_level`：制作档位，仅进入制作池时有值。
- `reference_type`：参考对象判断结果，待后续阶段写入。
- `adaptation_level`：剧本改编档位，待后续阶段写入。
- `performance_style`：最终演绎风格，待剧本改编阶段确认写入。
- `created_at`：项目创建时间。
- `updated_at`：项目最后更新时间。

### 主流程阶段枚举

`project_status` 只允许以下主流程阶段：

- `draft`
- `topic_scored`
- `reference_decided`
- `assets_checked`
- `design_ready`
- `script_ready`
- `storyboard_ready`
- `video_prompts_ready`
- `published`
- `reviewed`
- `archived`

### 生命周期枚举

`lifecycle_flag` 建议固定为以下值：

- `active`
- `blocked`
- `skipped`
- `abandoned`
- `completed`

## 中文显示与英文参数值规范

SceneForge 默认采用三层表达：

1. 对话层：中文优先
2. 摘要层：中文显示名 + 英文参数值
3. 协议层：英文参数值

执行要求如下：

- 所有结构化字段值继续使用英文参数值，作为唯一机器可读值。
- 所有 `summary` 和黑板概览默认使用中文描述；在关键状态、决策、档位、风格后建议追加英文参数值。
- 用户对话中不要只抛英文值，应优先写成“选题已评分（`topic_scored`）”这类形式。

推荐写法示例：

```yaml
summary: 该选题判断为进入制作（`go`），建议走夸张搞笑化（`exaggerated_comedy`）路线。
```

常用显示映射建议：

- `draft`：草稿
- `topic_scored`：选题已评分
- `reference_decided`：参考对象已确定
- `assets_checked`：资产已检查
- `design_ready`：设定已完成
- `script_ready`：剧本已完成
- `storyboard_ready`：分镜已完成
- `video_prompts_ready`：提示词已完成
- `published`：已发布
- `reviewed`：已复盘
- `archived`：已归档
- `active`：进行中
- `blocked`：阻塞中
- `skipped`：已跳过
- `abandoned`：已放弃
- `completed`：已完成
- `go`：进入制作
- `observe`：进入观察池
- `drop`：放弃
- `focus`：重点制作池
- `fast`：快速制作池
- `cinematic_serious`：正剧电影化
- `cinematic_comedy`：喜剧动画化
- `exaggerated_comedy`：夸张搞笑化
- `absurd_chaotic`：鬼畜离谱化

## 阶段补丁统一协议

每个阶段分区统一使用结构化 YAML 代码块，不再依赖 HTML 注释包围协议。

统一结构如下：

```yaml
patch_type:
version:
status:
updated_at:
summary:
data:
```

### 字段说明

- `patch_type`：阶段标识，例如 `scene-topic-gate`。
- `version`：补丁结构版本号，当前从 `1` 起。
- `status`：阶段状态。
- `updated_at`：补丁更新时间。
- `summary`：一句面向人的摘要，强制字段。
- `data`：阶段业务数据主体。

### 阶段状态枚举

`status` 固定为以下值之一：

- `pending`
- `in_progress`
- `completed`
- `blocked`
- `skipped`

## `PROJECT_BOARD.md` 模板建议

每个项目继续保留既有章节结构，但每章内部统一放一个 YAML 补丁块。

示例：

````md
## 1. 选题评分区

```yaml
patch_type: scene-topic-gate
version: 1
status: pending
updated_at:
summary: 待进行选题评分
data: {}
```
````

后续阶段统一沿用同一结构：

- `scene-reference-decider`
- `scene-asset-checker`
- `scene-design-builder`
- `scene-script-adapter`
- `scene-storyboard-director`
- `scene-video-prompt-builder`
- `scene-publish-review`

## `scene-topic-gate` 模块职责

`scene-topic-gate` 是整个流程的入口，职责限定如下：

### 负责内容

- 接收候选片段或桥段
- 按七维评分规则计算总分
- 产出进入制作、观察池或放弃池的判断
- 在进入制作时给出制作档位
- 记录后续阶段需要继承的风险提示和复用提示

### 不负责内容

- 参考对象裁定
- 资产复用命中判断
- 角色与场景设定
- 剧本改编
- 分镜设计
- 视频提示词生成

## `scene-topic-gate.data` 协议

`scene-topic-gate` 采用审计型 `data` 结构，字段如下：

```yaml
data:
  topic_name:
  source_material:
    source_type:
    source_name:
    source_locator:
    notes:
  total_score:
  performance_style_suggestion:
  production_level:
  decision:
  dimension_scores:
    热点价值:
    动画化适配度:
    改编空间:
    经典认知锚点:
    风险可控性:
    制作成本可控性:
    平台传播潜力:
  rationale:
  risk_notes:
  reuse_hints:
  evaluator_rule_version:
```

### 字段说明

- `topic_name`：候选片段名。
- `source_material`：来源结构体，为后续参考对象判断提供输入。
- `total_score`：选题总分。
- `performance_style_suggestion`：建议演绎风格，由选题阶段给出建议，不直接写入顶层。
- `production_level`：仅当 `decision = go` 时取值。
- `decision`：流程判断结果。
- `dimension_scores`：固定七维评分结果。
- `rationale`：评分依据列表。
- `risk_notes`：风险提示列表。
- `reuse_hints`：资产复用或系列化提示列表。
- `evaluator_rule_version`：评分规则版本号。

### `source_material` 字段口径

```yaml
source_material:
  source_type:
  source_name:
  source_locator:
  notes:
```

- `source_type`：建议枚举为 `original_work / specific_adaptation / hot_clip / user_input / mixed_reference`
- `source_name`：来源名，例如“86版西游记”
- `source_locator`：章节、集数、链接或关键词
- `notes`：补充说明

### `decision` 枚举

- `go`
- `observe`
- `drop`

### `production_level` 枚举

仅在 `decision = go` 时允许填写：

- `focus`
- `fast`

### `performance_style_suggestion` 枚举

- `cinematic_serious`
- `cinematic_comedy`
- `exaggerated_comedy`
- `absurd_chaotic`

### 七维评分字段固定

`dimension_scores` 固定使用以下字段：

- `热点价值`
- `动画化适配度`
- `改编空间`
- `经典认知锚点`
- `风险可控性`
- `制作成本可控性`
- `平台传播潜力`

## `scene-topic-gate` 状态推进规则

### 初始状态

- `project_status: draft`
- `next_stage: scene-topic-gate`
- `lifecycle_flag: active`

### 评分完成且 `decision = go`

- `project_status: topic_scored`
- `next_stage: scene-reference-decider`
- 顶层写入 `score`
- 顶层写入 `production_level`
- `lifecycle_flag: active`

### 评分完成且 `decision = observe`

- `project_status: topic_scored`
- `next_stage` 留空或等待人工判断
- 顶层写入 `score`
- `production_level` 留空
- `lifecycle_flag: active`

### 评分完成且 `decision = drop`

- `project_status: topic_scored`
- `next_stage` 留空
- 顶层写入 `score`
- `production_level` 留空
- `lifecycle_flag: abandoned`

## `scene-reference-decider` 模块职责

`scene-reference-decider` 负责把选题输入正式裁定成“本次创作到底按什么参考”的执行边界。

### 负责内容

- 基于 `scene-topic-gate` 的 `source_material` 判断参考对象类型
- 明确主参考和可选辅助参考
- 输出允许继承和禁止继承的边界
- 产出后续角色设计和剧本改编可直接消费的保留约束与避免约束
- 记录风险提示和下一步执行建议

### 不负责内容

- 角色与场景资产命中判断
- 角色外观具体设计
- 剧本改编正文
- 分镜与视频提示词生成

## `scene-reference-decider.data` 协议

`scene-reference-decider` 采用裁定型 `data` 结构，字段如下：

```yaml
data:
  reference_type:
  decision_summary:
  reference_boundary:
    primary_reference:
    secondary_reference:
    boundary_rule:
    allowed_inheritance:
    forbidden_inheritance:
  must_keep:
    - category:
      note:
  must_avoid:
    - category:
      note:
  risk_notes:
  next_action:
```

### 字段说明

- `reference_type`：本阶段最终裁定结果。
- `decision_summary`：一句短结论。
- `reference_boundary`：本次参考边界的结构化定义。
- `must_keep`：后续阶段必须保留的认知锚点或表达约束。
- `must_avoid`：后续阶段必须避免的表达约束。
- `risk_notes`：风险提示列表。
- `next_action`：进入下一阶段时的执行提示。

### `reference_type` 枚举

- `original_work`
- `specific_adaptation`
- `hybrid_reference`

说明：

- `source_material.source_type` 是输入来源描述。
- `reference_type` 是本阶段的最终裁定结果。
- “未定”不进入该业务枚举，由补丁 `status` 表达。

### `reference_boundary` 字段口径

```yaml
reference_boundary:
  primary_reference:
  secondary_reference:
  boundary_rule:
  allowed_inheritance:
  forbidden_inheritance:
```

- `primary_reference`：本次主参考对象。
- `secondary_reference`：可选辅助参考对象；没有则留空。
- `boundary_rule`：一句话说明本次按什么边界执行。
- `allowed_inheritance`：允许继承的内容分类列表。
- `forbidden_inheritance`：禁止继承的内容分类列表。

### 固定继承分类

`allowed_inheritance` 和 `forbidden_inheritance` 只允许从以下固定分类中选择：

- `角色身份`
- `角色关系`
- `服装轮廓`
- `标志道具`
- `情绪核心`
- `剧情骨架`
- `世界观母题`
- `场景气质`
- `镜头动势`

### `must_keep` 与 `must_avoid` 结构

`must_keep` 和 `must_avoid` 使用“分类 + 说明”的轻结构：

```yaml
must_keep:
  - category:
    note:

must_avoid:
  - category:
    note:
```

- `category`：复用固定继承分类。
- `note`：结合本次项目语境写明具体保留或规避要求。

## `scene-reference-decider` 状态推进规则

### 初始状态

- `project_status: topic_scored`
- `next_stage: scene-reference-decider`
- `lifecycle_flag: active`

### 裁定完成

- `project_status: reference_decided`
- `next_stage: scene-asset-checker`
- 顶层写入 `reference_type`
- `lifecycle_flag: active`

### 阻塞场景

- 仅当参考边界存在真实冲突、无法继续执行时，补丁 `status` 才使用 `blocked`
- 顶层 `blocker_note` 只在真正阻塞时写入
- 普通风险提醒只留在 `risk_notes`

### 关于 `hybrid_reference`

- `hybrid_reference` 属于正常业务结果，不自动触发阻塞
- 只要 `reference_boundary` 写清楚，补丁 `status` 可以为 `completed`
- 是否阻塞取决于边界是否可执行，而不是是否为混合参考

## `scene-reference-decider` 示例

````md
## 2. 参考对象判断区

```yaml
patch_type: scene-reference-decider
version: 1
status: completed
updated_at: 2026-06-01
summary: 本次以86版西游记为主参考，允许继承母题级角色和剧情骨架，但避免直接复刻具体表演与镜头。
data:
  reference_type: hybrid_reference
  decision_summary: 以86版西游记为主参考，辅以西游记母题边界，按“高认知锚点保留、具体影视表达降继承”执行。
  reference_boundary:
    primary_reference: 86版西游记
    secondary_reference: 西游记原著母题
    boundary_rule: 角色身份、关系和剧情骨架优先继承母题与大众认知，避免直接复刻86版具体镜头和表演表达。
    allowed_inheritance:
      - 角色身份
      - 角色关系
      - 标志道具
      - 情绪核心
      - 剧情骨架
      - 世界观母题
    forbidden_inheritance:
      - 镜头动势
      - 服装轮廓
  must_keep:
    - category: 角色身份
      note: 必须保留孙悟空、唐僧、白骨精之间的基础身份关系和冲突结构。
    - category: 情绪核心
      note: 必须保留“忠诚与误解并存”的核心戏剧张力。
    - category: 剧情骨架
      note: 必须保留白骨精三次试探与识破的桥段骨架。
  must_avoid:
    - category: 镜头动势
      note: 避免直接复刻86版的经典机位切换、走位和表演调度。
    - category: 服装轮廓
      note: 避免过度贴近86版真人戏服造型，改为动画电影化转译。
  risk_notes:
    - 若角色外观参考强度过高，容易滑向具体影视版本复刻。
    - 若对白保留过多，后续剧本改编阶段需重新评估风险。
  next_action: 进入 scene-asset-checker，优先判断孙悟空角色资产和山林、妖洞类场景资产是否已有可复用版本。
```
````

## 当前未覆盖项

以下内容后续需要单独设计，但不阻塞本协议生效：

1. 观察池项目的长期回流机制
2. 顶层空值的最终书写规范，例如空字符串或 `null`

## 结论

本协议作为 SceneForge 第一版黑板基线，先保证“总控可路由、子 Skill 可统一回写、设计可持续扩展”。

后续新增子 Skill 时，默认必须继承本文档的顶层字段规则和阶段补丁规则，除非有新的协议升级文档明确替代。
