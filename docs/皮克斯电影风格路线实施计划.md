# 皮克斯电影风格路线实施计划

## 文档目的

本文档用于跟踪 SceneForge 当前阶段的 `pixar_like` 动画电影风格路线落地工作。

当前目标不是扩展多风格平台，而是先把第一套动画电影导演链路跑通、跑透。

核心路线：

```text
先跑通 pixar_like
↓
沉淀通用导演协议
↓
再抽象多风格 Style Profile
```

---

# 一、当前实施范围

本阶段只聚焦 `pixar_like` 路线增强。

重点增强：

- Story Beat：叙事节拍层
- Performance Director：表演导演 Skill
- Storyboard Director v2：导演意图型分镜
- Audio Director：声音导演 Skill
- Video Prompt Builder v2：融合表演、分镜、声音的最终视频提示词
- 三层时间模型：Story Beat / Shot / Segment

不在本阶段展开：

- 多风格 Style Profile 目录
- scene-style-selector
- DreamWorks-like、Anime cinematic、Noir 等其他风格包

---

# 二、实施原则

## 1. 不破坏现有链路

当前链路已经可用，不做大范围重构。

采用渐进式补丁：

```text
新增 Skill
↓
更新 output-contract
↓
更新阶段推进规则
↓
后续再跑样例项目验证
```

## 2. 黑板仍保持薄索引

`PROJECT_BOARD.md` 顶层只保留跨阶段字段。

长内容继续落盘到：

```text
projects/<project>/details/
projects/<project>/outputs/
```

## 3. Pixar-like 只作为内部风格路线

对外生成 Prompt 避免直接复刻具体公司、电影、角色、音乐。

推荐表达：

```text
高品质 3D 动画电影质感
温暖角色驱动的动画电影表演
电影级动画角色与镜头语言
```

避免表达：

```text
完全像某部 Pixar 电影
复刻某个具体 Pixar 角色
使用某部电影的主题音乐
```

# 四、核心设计增强

## 1. Story Beat 层

`scene-script-adapter` 不再只输出普通 `scene_beats`，而要输出更清晰的 `story_beats`。

关系：

```text
Story Beat = 叙事情绪节拍
Shot = 导演镜头表达单元
Segment = 视频生成技术切片
```

正确结构：

```text
Story Beat
  -> Shot
    -> Segment
      -> Final Video
```

## 2. Performance Director

新增表演导演阶段，解决：

- 角色眼神
- 微表情
- 肢体动作
- 停顿
- 反应节奏
- 角色标志性动作

输出：

```text
details/performance_sheet_v*.md
```

## 3. Storyboard Director v2

分镜阶段需要继承：

- story_beats
- performance_sheet
- design summary
- reference boundary

新增字段：

- `camera_intent`
- `acting_intent`
- `sound_intent`
- `emotional_goal`
- `covered_beats`
- `continuity_in`
- `continuity_out`

## 4. Audio Director

新增声音导演阶段，解决：

- 配音方向
- 拟音设计
- 环境音
- 静默点
- 配乐主题
- 音频混合计划

输出：

```text
details/audio_plan_v*.md
outputs/audio/music_prompt_v*.md
outputs/audio/foley_prompt_v*.md
outputs/audio/audio_mix_plan_v*.md
```

## 5. Video Prompt Builder v2

最终视频提示词阶段必须整合：

- 角色设定
- 表演设计
- 分镜方案
- 声音方案
- 音乐提示
- 拟音提示
- Segment 拼接规则
