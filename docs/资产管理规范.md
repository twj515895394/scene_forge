# 资产管理规范

## 资产库职责

`assets/` 保存长期可复用资产，不保存单次项目产物。

## 资产分类

- `assets/characters/`：角色资产。
- `assets/scenes/`：场景资产。
- `assets/props/`：核心系列道具。普通道具不入库。

## 强制复用判断

任何项目进入制作后，必须先做：

1. 角色资产复用判断。
2. 场景资产复用判断。

道具资产判断为轻量备选：只有满足角色识别、剧情动作、系列复用任意一项的道具才进入判断。

## 版本机制

角色、场景、核心系列道具一旦入库，必须带版本号。

命名建议：

```text
资产类型_资产名称_v主版本.子版本_变体说明
```

示例：

```text
角色_孙悟空_v1.0_经典锚点版.md
场景_山林小路_v1.0_黄昏神秘版.md
道具_金箍棒_v1.0_经典识别版.md
```

## 版本号含义

- `v1.0`：首次可用版本。
- `v1.1 / v1.2`：轻微调整，不改变核心识别。
- `v2.0`：重大重设计。
