# 协议层与主链路设计基线

## 文档目的

本文档用于冻结 SceneForge v3 的黑板协议、主链路路由原则和通用项目记忆字段。

目标：

1. 让总控 Skill `scene-forge` 稳定读取项目状态并只执行当前 `next_stage`。
2. 让所有子 Skill 后续按同一套补丁协议回写 `PROJECT_BOARD.md`。
3. 把样例项目暴露的问题沉淀为通用 SOP / Skill / PROJECT_BOARD 记忆字段，而不是把样例角色、台词、站位或桥段固化到通用协议中。
4. 控制执行阶段的上下文读取边界和 Token 成本。

## 适用范围

- `projects/*/PROJECT_BOARD.md` 顶层 YAML 索引字段
- 各阶段分区补丁结构
- `scene-forge` 总控路由原则
- `scene-topic-gate`
- `scene-reference-decider`
- 显示规范与主链路状态推进口径
- v3 通用记忆字段：确认闸门、分段策略、Hero Moment、Bridge Shot、道具状态机、Blocking Map、Faction Layout

细粒度输出协议以各 Skill 的 `references/output-contract.md` 为准。

## v3 设计原则

1. 顶层 YAML 只保留跨阶段共用的薄索引字段和少量通用记忆字段。
2. 各阶段细节只落在对应分区补丁里，避免顶层和分区双写漂移。
3. 补丁格式优先机器可解析，同时保留最小的人类可读摘要。
4. 主状态机只表达主流程阶段，异常态单独建模。
5. 总控只能执行 `PROJECT_BOARD.md` 的 `next_stage`，不得跳步或连跑多个阶段。
6. 用户说“继续”只能解释为执行当前 `next_stage`。
7. 用户纠错、补充偏好或指出问题，不等于授权落盘。
8. 剧本、设计、分镜、视频提示词阶段必须先给方案预览，用户确认后再落盘正式文件。
9. SceneForge 当前只输出提示词、制作说明和发布文案，不生成图片、视频或音频。
10. 执行阶段默认采用 `compact` Token 预算，不扫描 `docs/`、`.handoff/` 或历史项目输出。

## `PROJECT_BOARD.md` 顶层协议

顶层 YAML 使用薄索引 + 通用记忆字段模式：

```yaml
---
project_name:
project_status:
next_stage:
lifecycle_flag:
blocker_note:
score:
production_level:
reference_type:
adaptation_level:
performance_style:
target_total_duration_seconds:
segment_duration_seconds:
context_policy:
user_confirmations:
segment_strategy:
hero_moments:
bridge_shots:
prop_state_machines:
blocking_map:
faction_layout:
created_at:
updated_at:
---
```

### 字段说明

- `project_name`：项目名。
- `project_status`：当前主流程阶段。
- `next_stage`：下一建议阶段，供总控 Skill 路由使用；总控只能执行该阶段。
- `lifecycle_flag`：项目生命周期标记。
- `blocker_note`：阻塞原因摘要；无阻塞时留空。
- `score`：选题总分，全局索引值。
- `production_level`：制作档位，仅进入制作池时有值。
- `reference_type`：参考对象判断结果。
- `adaptation_level`：剧本改编档位。
- `performance_style`：最终演绎风格。
- `target_total_duration_seconds`：整条视频目标总时长。
- `segment_duration_seconds`：单个视频生成片段时长，通常为 `10` 或 `15`，不等于整片时长。
- `context_policy`：上下文读取边界和 Token 预算策略。
- `user_confirmations`：关键创作决策的用户确认状态。
- `segment_strategy`：整片目标时长、单段时长、段数和分段模式。
- `hero_moments`：当前项目自己的看点镜头或记忆点。
- `bridge_shots`：Segment 之间的桥接分镜。
- `prop_state_machines`：核心道具状态机。
- `blocking_map`：角色空间调度图。
- `faction_layout`：角色阵营布局。
- `created_at`：项目创建时间。
- `updated_at`：项目最后更新时间。

### 建议初始化默认值

```yaml
context_policy:
  mode: minimal
  allow_docs_scan: false
  active_protocol_docs: []
  forbidden_by_default:
    - docs/
    - .handoff/
    - 会话记录_*.md
    - 历史项目输出
  token_budget:
    default_stage_budget: compact
    require_reason_for_extra_reads: true

user_confirmations:
  duration_strategy_confirmed: false
  script_plan_confirmed: false
  design_direction_confirmed: false
  storyboard_plan_confirmed: false
  video_prompt_plan_confirmed: false

segment_strategy:
  target_total_duration_seconds:
  segment_duration_seconds: 10
  segment_count:
  segmentation_mode: fixed_10s
  confirmation_note:

hero_moments: []
bridge_shots: []
prop_state_machines: []
blocking_map:
  spatial_axis:
  zones: []
  continuity_rule:
faction_layout:
  factions: []
```

- `draft`
- `topic_scored`
- `reference_decided`
- `assets_checked`
- `design_ready`
- `script_ready`
- `performance_ready`
- `storyboard_ready`
- `audio_ready`
- `video_prompts_ready`
- `published`
- `reviewed`
- `archived`

## 生命周期枚举

`lifecycle_flag` 建议固定为以下值：

- `active`
- `blocked`
- `skipped`
- `abandoned`
- `completed`

## 中文显示与英文参数值规范

SceneForge 默认采用三层表达：

1. 对话层：中文优先
2. 摘要层：中文显示名 + 英文参数值
3. 协议层：英文参数值

执行要求如下：

- 所有结构化字段值继续使用英文参数值，作为唯一机器可读值。
- 所有 `summary` 和黑板概览默认使用中文描述；在关键状态、决策、档位、风格后建议追加英文参数值。
- 用户对话中不要只抛英文值，应优先写成“选题已评分（`topic_scored`）”这类形式。

常用显示映射建议：

- `draft`：草稿
- `topic_scored`：选题已评分
- `reference_decided`：参考对象已确定
- `assets_checked`：资产已检查
- `design_ready`：设定已完成
- `script_ready`：剧本已完成
- `performance_ready`：表演设计已完成
- `storyboard_ready`：分镜已完成
- `audio_ready`：声音方案已完成
- `video_prompts_ready`：提示词已完成
- `published`：已发布
- `reviewed`：已复盘
- `archived`：已归档
- `active`：进行中
- `blocked`：阻塞中
- `skipped`：已跳过
- `abandoned`：已放弃
- `completed`：已完成
- `go`：进入制作
- `observe`：进入观察池
- `drop`：放弃
- `focus`：重点制作池
- `fast`：快速制作池
- `cinematic_serious`：正剧电影化
- `cinematic_comedy`：喜剧动画化
- `exaggerated_comedy`：夸张搞笑化
- `absurd_chaotic`：鬼畜离谱化

## 阶段补丁统一协议

每个阶段分区统一使用结构化 YAML 代码块。

统一结构如下：

```yaml
patch_type:
version:
status:
updated_at:
summary:
data:
```

### 字段说明

- `patch_type`：阶段标识，例如 `scene-topic-gate`。
- `version`：补丁结构版本号。
- `status`：阶段状态。
- `updated_at`：补丁更新时间。
- `summary`：一句面向人的摘要，强制字段。
- `data`：阶段业务数据主体。

### 阶段状态枚举

`status` 固定为以下值之一：

- `pending`
- `in_progress`
- `completed`
- `blocked`
- `skipped`

## 阶段确认闸门

以下阶段默认必须先给方案预览，用户确认后才允许落盘正式文件或推进阶段：

- `scene-script-adapter`：确认总时长、分段策略、剧本方案和 Story Beat 方向。
- `scene-design-builder`：确认角色方向、场景道具清单、统一视觉语言和参考强度。
- `scene-storyboard-director`：确认分镜结构、Hero Shot、Bridge Shot、Segment Plan 和 Blocking Map。
- `scene-video-prompt-builder`：确认分段提示词结构、参考图使用方案和连续性策略。

## 上下文读取边界与 Token 预算

执行阶段默认采用：

```yaml
token_budget_level: compact
context_policy:
  mode: minimal
  allow_docs_scan: false
```

默认读取：

```text
projects/<project>/PROJECT_BOARD.md
.agents/skills/<current-skill>/SKILL.md
.agents/skills/<current-skill>/references/output-contract.md
当前阶段明确依赖的输入文件
```

默认不读取：

```text
docs/
.handoff/
会话记录_*.md
历史项目输出
其他无关项目目录
```

如果需要超出默认读取范围，必须满足至少一个条件：

1. 用户明确要求分析、复盘、重构协议或读取相关文档。
2. `PROJECT_BOARD.md` 的 `context_policy.active_protocol_docs` 明确列出该文档。
3. 当前阶段无法用最小上下文完成，且回复中说明额外读取原因。

预算等级：

- `compact`：只读黑板、当前 Skill、当前输出协议和必要输入文件。
- `standard`：允许读取 1-3 个明确依赖文件。
- `deep`：仅用于复盘、协议改造、全链路分析或用户明确要求的深度读取。

## `scene-topic-gate` 模块职责

`scene-topic-gate` 是整个流程的入口，职责限定如下：

### 负责内容

- 接收候选片段或桥段
- 按七维评分规则计算总分
- 产出进入制作、观察池或放弃池的判断
- 在进入制作时给出制作档位
- 记录后续阶段需要继承的风险提示和复用提示

### 不负责内容

- 参考对象裁定
- 资产复用命中判断
- 角色与场景设定
- 剧本改编
- 分镜设计
- 视频提示词生成

## `scene-topic-gate.data` 协议

`scene-topic-gate` 采用审计型 `data` 结构，字段如下：

```yaml
data:
  topic_name:
  source_material:
    source_type:
    source_name:
    source_locator:
    notes:
  total_score:
  performance_style_suggestion:
  production_level:
  decision:
  dimension_scores:
    热点价值:
    动画化适配度:
    改编空间:
    经典认知锚点:
    风险可控性:
    制作成本可控性:
    平台传播潜力:
  rationale:
  risk_notes:
  reuse_hints:
  evaluator_rule_version:
```

### `decision` 枚举

- `go`
- `observe`
- `drop`

### `production_level` 枚举

仅在 `decision = go` 时允许填写：

- `focus`
- `fast`

### `performance_style_suggestion` 枚举

- `cinematic_serious`
- `cinematic_comedy`
- `exaggerated_comedy`
- `absurd_chaotic`

## `scene-reference-decider` 模块职责

`scene-reference-decider` 负责把选题输入正式裁定成“本次创作到底按什么参考”的执行边界。

### 负责内容

- 基于 `scene-topic-gate` 的 `source_material` 判断参考对象类型
- 明确主参考和可选辅助参考
- 输出允许继承和禁止继承的边界
- 产出后续角色设计和剧本改编可直接消费的保留约束与避免约束
- 记录风险提示和下一步执行建议

### 不负责内容

- 角色与场景资产命中判断
- 角色外观具体设计
- 剧本改编正文
- 分镜与视频提示词生成

## `scene-reference-decider.data` 协议

```yaml
data:
  reference_type:
  decision_summary:
  reference_boundary:
    primary_reference:
    secondary_reference:
    boundary_rule:
    allowed_inheritance:
    forbidden_inheritance:
  must_keep:
    - category:
      note:
  must_avoid:
    - category:
      note:
  risk_notes:
  next_action:
```

### `reference_type` 枚举

- `original_work`
- `specific_adaptation`
- `hybrid_reference`

### 固定继承分类

`allowed_inheritance` 和 `forbidden_inheritance` 建议从以下分类中选择：

- `角色身份`
- `角色关系`
- `服装轮廓`
- `标志道具`
- `情绪核心`
- `剧情骨架`
- `世界观母题`
- `场景气质`
- `镜头动势`
- `空间关系`
- `道具状态`

## 状态推进规则

### 初始状态

- `project_status: draft`
- `next_stage: scene-topic-gate`
- `lifecycle_flag: active`

### 评分完成且 `decision = go`

- `project_status: topic_scored`
- `next_stage: scene-reference-decider`
- 顶层写入 `score`
- 顶层写入 `production_level`
- `lifecycle_flag: active`

### 评分完成且 `decision = observe`

- `project_status: topic_scored`
- `next_stage` 留空或等待人工判断
- 顶层写入 `score`
- `production_level` 留空
- `lifecycle_flag: active`

### 评分完成且 `decision = drop`

- `project_status: topic_scored`
- `next_stage` 留空
- 顶层写入 `score`
- `production_level` 留空
- `lifecycle_flag: abandoned`

### 裁定完成

- `project_status: reference_decided`
- `next_stage: scene-asset-checker`
- 顶层写入 `reference_type`
- `lifecycle_flag: active`

### 阻塞场景

- 仅当参考边界存在真实冲突、无法继续执行时，补丁 `status` 才使用 `blocked`
- 顶层 `blocker_note` 只在真正阻塞时写入
- 普通风险提醒只留在 `risk_notes`

## 输出边界

SceneForge 当前只负责输出提示词、制作说明、结构化方案和发布文案。

禁止 Agent 使用：

```text
我已经生成了角色图。
我已经生成了故事板图。
我已经生成了视频。
我后台调用绘图引擎运行了提示词。
我已经把生成图片嵌入文档。
```

推荐表述：

```text
已生成用于制作角色设定图的提示词。
已生成用于制作故事板图的提示词。
已生成用于基于角色图、场景道具图、故事板图生成视频的分段提示词。
这些图片和视频需要用户在外部平台手动生成。
```
