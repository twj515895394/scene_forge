# scene-publish-review

负责 SceneForge 流程的最后一段：把创作结果整理成可发布内容，并在发布后做复盘与资产沉淀建议。

## 何时使用

在以下场景使用此技能：

- 总控 Skill 发现项目 `next_stage` 为 `scene-publish-review`
- 已完成故事板和视频提示词，需要整理平台发布内容
- 需要记录发布后表现，并判断哪些角色、场景或道具应沉淀回资产库

如果提示词和素材准备还没完成，不要优先使用本技能。

## 执行步骤

1. 读取项目 `PROJECT_BOARD.md`，确认 `project_status` 已到 `video_prompts_ready`。
2. 读取提示词输出、项目定位和参考边界，确认发布表达不要越界。
3. 读取 `references/output-contract.md`，确认发布物分类、复盘字段和落盘路径。
4. 生成标题、封面文案、平台发布文案、评论区引导、字幕或配音文案。
5. 将最终可发布文本写入 `outputs/publish_copy/`；黑板里只保留摘要、版本和路径。
6. 在发布后补充数据复盘摘要，并给出资产沉淀建议。
7. 输出单个 YAML 补丁块；`summary` 使用中文，必要时附英文参数值。
8. 将状态推进建议交回总控 Skill，推进到 `published` 或 `reviewed`。

## 关键规则

- 输出内容至少包括：标题、封面文案、平台发布文案、评论区引导、字幕/配音文案、发布后数据复盘、资产沉淀建议
- 发布表达必须服从参考边界和风险边界，不直接强化敏感 IP 绑定措辞
- 黑板只保留发布摘要、复盘状态和文件路径，不直接塞完整发布正文
