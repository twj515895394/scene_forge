# scene-design-builder

负责角色与场景的动画电影化设定。根据制作档位和资产判断结果，决定输出轻量锁定卡还是完整设定，并同步产出可直接生成角色图/场景图的 prompt 文件。

## 何时使用

在以下场景使用此技能：

- 总控 Skill 发现项目 `next_stage` 为 `scene-design-builder`
- 已完成资产复用判断，需要为角色和场景生成本次项目可执行的视觉设定
- 需要把参考强度、动画电影化风格和资产复用结论合并成统一设计结果

如果资产命中判断还没完成，或已经进入剧本改编，不要优先使用本技能。

## 执行步骤

1. 读取项目 `PROJECT_BOARD.md`，确认 `project_status` 已到 `assets_checked`。
2. 读取 `scene-reference-decider` 的边界和 `scene-asset-checker` 的复用结论。
3. 读取 `references/output-contract.md`，确认制作档位、参考强度和输出落点。
4. 判断本次走“轻量锁定卡”还是“完整设定”路径。
5. 先定义本次项目的统一视觉语言基线，包括形状语言、剪影锚点、比例策略、材质策略、色彩剧本和场景设计化规则。
6. 为角色、场景和关键道具分别产出统一的锁定摘要、风格约束和复用说明；三者都必须继承同一套视觉语言基线。
7. 产出可直接生成设定图的 prompt 文件，写入 `outputs/design_prompts/`；prompt 必须明确要求“角色设定板”或“环境设定板”，而不是单张海报式图片。
8. 对较长的角色/场景锁定卡正文，写入 `details/`；黑板里只保留摘要和引用路径。
9. 输出单个 YAML 补丁块，并给出下一阶段的剧本改编注意事项。
10. 将状态推进建议交回总控 Skill，进入 `scene-script-adapter`。

## 关键规则

- 重点制作池优先产出完整角色设定和完整场景设定
- 快速制作池优先产出轻量角色锁定卡和轻量场景锁定卡
- 角色参考强度分为：高锚点参考型 / 中度参考型 / 低参考原创转译型
- 生成 prompt 允许使用强风格锚词，例如“皮克斯电影级动画风格”“超高质量 3D 角色动画”“电影级灯光与镜头语言”；但发布文案和对外包装仍尽量避免品牌绑定词
- 输出必须优先服务生成链路：拿到 prompt 文件就能直接生成角色设定图和场景设定图
- 角色设定图默认要求包含多视角、表情区、动作姿态区、服装配件细节区，不能退化成单张站姿人像
- 任何角色都必须先过一轮动画角色设计判断：形状语言、头身比、剪影锚点、眼睛策略、表情系统、材质方向，不能只堆服装和年龄描述
- 任何场景和关键道具也必须过一轮统一视觉语言判断：不能只有人物像动画电影，场景和道具却像普通背景图或商品图
- 本技能输出的核心不只是“某个角色长什么样”，而是“这一整个视觉宇宙遵循什么电影级设计语言”
