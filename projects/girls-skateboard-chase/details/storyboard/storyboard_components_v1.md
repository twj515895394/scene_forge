# 故事板架构组件与质量检查 (Storyboard Components & Quality Check) - v1.0

本文件为 `girls-skateboard-chase` 项目的 3D 动画分镜架构组件，包含 Beat Skeleton、Content Breakdown、VGU 规划、动作/情绪交接连续性设计以及故事板质量自检，供后续视频 Prompt 组装使用。

---

## 1. 节拍骨架与内容拆分 (Beat Skeleton & Content Breakdown)

*   **B01: 起跑线蓄力 (VGU-01, 镜头 01-04)**
    *   *节拍目的*：引入人物与拼色潮服，建立起跑动能。
    *   *动作与情绪流*：站姿静态对峙 -> 对白挑衅击掌 -> 起跑点板蹬地爆发。
    *   *空间转移*：Zone A (山顶起点)。
    *   *镜头数建议*：4 个镜头。
    *   *内容拆分 (Breakdown)*：
        *   `setup`: 山顶开阔柏油路，逆光两个S型剪影 (S01)。
        *   `dialogue`: 女孩 A 大喊吃冰挑战，女孩 B 抿嘴角自信微笑 (S02)。
        *   `action`: 两人凌空击掌，画面定格 0.2s 增强张力 (S03)。
        *   `transition`: 哨声响，滑板鞋蹬地起跑，轮子狂转，尘土飞扬 (S04)。
*   **B02: 急弯漂移博弈 (VGU-02, 镜头 05-07)**
    *   *节拍目的*：表现山道速降重力感与弯道趣味反超。
    *   *动作与情绪流*：A 贴地手套平拍滑行漂移 -> B 内线死角切入反超 -> B 超车 Wink 做鬼脸，A 气鼓鼓震惊。
    *   *空间转移*：Zone B (S弯区)。
    *   *镜头数建议*：3 个镜头。
    *   *内容拆分 (Breakdown)*：
        *   `action`: 女孩 A 45度倾斜，手掌擦地冒烟漂移 (S05)。
        *   `reveal`: 女孩 B 内线切入超车，板轮贴路肩疾驰 (S06)。
        *   `contrast_payoff`: 女孩 B 偏头 wink 吐舌做鬼脸，女孩 A 气急败坏震惊吃瘪 (S07)。
*   **B03: Ollie飞跃障碍 (VGU-03, 镜头 08-11)**
    *   *节拍目的*：展现高难特技动作与空中撞板默契。
    *   *动作与情绪流*：直道前现塑料墩与落叶障区 -> B Ollie飞跃落叶，A Kickflip翻板绕墩 -> 半空 Board-tap 碰板尾定格 -> 落地 Squash 弹性缓冲。
    *   *空间转移*：Zone C (施工障区)。
    *   *镜头数建议*：4 个镜头。
    *   *内容拆分 (Breakdown)*：
        *   `setup`: 交代隔离墩与落叶障区地标，女孩 B 冲刺起跳 (S08)。
        *   `stylized_impact`: 女孩 B 抱膝 Ollie，女孩 A 排水渠边缘 Kickflip 翻板 (S09)。
        *   `hero_moment`: 半空两板板尾精准相撞，Clack 脆响，慢镜头 (S10)。
        *   `recovery`: 落地弹性缓冲，重回公路 (S11)。
*   **B04: 气流牵引拉锯 (VGU-04, 镜头 12-14)**
    *   *节拍目的*：展现战术减阻超车与身体蹭碰。
    *   *动作与情绪流*：A 弓身呈水滴流线型贴在 B 气流牵引区 -> B 回头吐舌，A 右切变道蹭肩超车 -> A 坏笑领先，B 宠溺无奈笑。
    *   *空间转移*：Zone C (崎岖直道)。
    *   *镜头数建议*：3 个镜头。
    *   *内容拆分 (Breakdown)*：
        *   `action`: 极低俯拍，女孩 A 呈水滴流线弓身风阻姿态 (S12)。
        *   `stylized_impact`: 女孩 B 回头，女孩 A 超车肩膀轻轻碰撞 (S13)。
        *   `reaction`: 女孩 A 马尾掠过镜头前，女孩 B 无奈宠溺一笑 (S14)。

## 3. 故事板质量自查 (Storyboard Quality Check)

在输出故事板提示词前，我们对本分镜方案进行了 5 项硬性质量检查：

1.  **镜头职责自查 (Shot Role)**：20 个镜头均分配了具体的戏剧功能（如 S01 setup, S07 contrast_payoff, S10 hero_moment），确保没有冗余镜头。
2.  **动作交接自查 (Action Handoff)**：超车进出方向（如 S06 强切内弯出画右，S07 承接 B 领先 A 吃瘪）、起跳落地缓冲（S09 飞跃 -> S10 碰板尾 -> S11 落地 Squash）均完成了运动物理链闭环。
3.  **空间交接自查 (Spatial Handoff)**：严格以赛道双黄线为轴。S05 机位在弯内，S16 并漂机位保持在内弯仰拍，完全规避了 180 度越轴镜像翻转错误。
4.  **连续性四线自查 (Four-line Continuity)**：
    *   *Rhythm 线*：竞速快切与特技 Hold / slow-motion 结合；
    *   *Action 线*：板面、护具在各段的相对位置一致；
    *   *Emotion 线*：眼神对焦链闭环（起跑、超车、腾空、蹭肩、碰拳时两人生死对焦）；
    *   *Space 线*：松树、红白护栏、施工锥筒、格子线草坪空间地标顺序出现。
5.  **双版一致性自查 (Dual-prompt Consistency)**：稍后输出的故事板 Prompts 中，控制版与风格版使用同源镜头骨架，仅在生成控制层与画面美术调性层进行修饰，保证生图时结构互通。

> [!TIP]
> **免责与执行期忠告**：
> 故事板 Blocking 规划、镜头语言描述和连续性约束能够有效降低 AI “抽卡”的漂移概率，但并不能完全保证视频生成模型一次性稳定产出。建议在生成时，首选 **Flux.1 Dev** 渲染设定图作为 Base 图，再将画面喂给视频生成模型进行动作补帧。
