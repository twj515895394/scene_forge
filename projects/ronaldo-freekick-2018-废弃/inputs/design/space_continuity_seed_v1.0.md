# 空间连续性种子规范 (Space Continuity Seed) v1.0

本规范用于保障在后续分镜设计（storyboard）及视频提示词生成（video prompts）中，跨镜头的空间方位、轴线朝向及位置连贯性。

---

## 1. 空间坐标系定义 (Spatial Coordinate System)

以罚球点（大禁区弧顶外白点）作为局部空间坐标系原点 $O(0, 0, 0)$：

*   **X 轴 (横向轴线)**：平行于球门线，面向球门方向时，正方向指向**右侧角旗区**，负方向指向左侧角旗区。
*   **Y 轴 (高度轴线)**：正方向指向**天空**，负方向指向地下。
*   **Z 轴 (纵向深度)**：正方向指向**球门中心**，负方向指向中圈方向。

---

## 2. 空间锚点与关键地标 (Anchor Spaces & Landmarks)

后续所有镜头（Shots）必须以此 3 个标志性空间锚点建立连续性对齐：

1.  **罚球锚点 (`Anchor_O`)**：原点 $O(0, 0, 0)$ 处。
    *   *视觉地标*：地面上有皮球、裁判画下的白色泡沫喷雾弧线。
    *   *交互角色*：主罚者“总裁”。
2.  **人墙防守锚点 (`Anchor_Wall`)**：坐标约 $(0, 0, 9.15)$ 处。
    *   *视觉地标*：草皮上横向平行的喷雾直线白痕。
    *   *交互角色*：由 5 名身着白色西班牙队服防守球员组成的“白色人墙”。
3.  **球门防守锚点 (`Anchor_Goal`)**：坐标约 $(0, 0, 20)$ 至 $(0, 0, 25)$。
    *   *视觉地标*：由白漆门柱与挂网球门框包围的矩形空间。
    *   *交互角色*：守门员“鸭哥”、柔性球网。
4.  **庆祝落地锚点 (`Anchor_SIUUU`)**：坐标约 $(15, 0, 23)$，即球门右侧角旗区。
    *   *视觉地标*：黄色的角旗杆与角旗。
    *   *交互角色*：“总裁”SIUUU 落地定格点。

---

## 3. 轴线保护规则 (Axis & Staging Rules)

为防止镜头产生方向性混乱，在动作前半段和击球段，必须严格遵守以下关系：

*   **180度射门轴线 (Shooting Axis Line)**：由原点指向球门中心的轴线。
    *   *规则一*：主摄像机机位必须始终保持在轴线的**右侧（X 轴正向半区）**。也就是说，镜头中葡萄牙进攻方向（球门方向）应始终在画面的**右侧**，西班牙防守方向在**左侧**。
    *   *规则二*：总裁在罚球备战时的正脸及侧脸特写，镜头朝向应为 **Z 轴正向**（偏 X 正向）的微距跟焦，确保眼神方向明确指向右前方的球门。
*   **越墙弧线子弹时间轴线**：
    *   从人墙后侧面向总裁视角（Z 轴负向）拍摄，足球应从画面左下向右上划过人墙头顶，展现电梯球剧烈下坠轨迹。
*   **回放阶段轴线转换说明**：
    *   Beat 5 的多角度回放虽然会切换机位（如球门后视角），但每一个切换必须显式声明“球门后视角的反向视线”以防混淆。

---

## 4. 角色进出空间逻辑 (Entrance & Exit Logic)

*   **足球**：
    *   从主罚点起动，沿抛物线轨迹运动，越过 `Anchor_Wall`，击中 `Anchor_Goal` 的右上死角球网，随后在重力下垂直下坠跌落至草坪静止。
*   **“总裁”**：
    *   B1-B4 始终在 `Anchor_O` 的 3 米半径内活动。
    *   进球后（B5），沿着 X 轴正向偏 Z 正向斜线狂奔，切入 `Anchor_SIUUU`（右侧角旗区）完成飞奔、跃起、转体 180 度，最终在角旗杆旁双脚着地定格。
*   **“鸭哥”**：
    *   全程在 `Anchor_Goal` 的球门线前做横向平移，扑救动作指向球门右上角死角，扑救落空后落地静止。
