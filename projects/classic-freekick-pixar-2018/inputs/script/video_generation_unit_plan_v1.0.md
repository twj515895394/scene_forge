# 视频生成单元规划书 (Video Generation Unit Plan) v1.0

本规划书将 40 秒的故事剧本拆分为 4 个视频生成单元（Video Generation Unit, VGU），每个单元时长约 10 秒。本设计旨在为后续的视频生成模型提供结构化的切分边界，确保视频剪辑拼接时的时空与物理连续性。

---

## 1. 视频生成单元明细 (VGU Breakdown)

### VGU 1：赛况对峙与起步助跑 (TIME: 0.0s - 10.0s)
*   **承载节拍**：B01 (比分 HUD)、B02 (总裁备战)、B03 (哨响助跑)
*   **镜头内容**：从夜空中霓虹比分大屏 HUD (`RED 2 - 3 WHT`，`TIME: 88:00`) 快切至总裁正面深呼吸、拉裤脚特写，最后以裁判鸣哨、总裁低头俯冲起跑的极速动态结束。
*   **动作连续性控制点**：
    *   B02 总裁跨立叉腰深呼吸的动作惯性，在拉裤脚动作时大腿肌肉保持紧绷状态；
    *   B03 总裁起跑的俯冲重心必须保持向前，且球鞋在草地上蹬起的泥土颗粒需在 10.0s 时刻飞扬至半空。
*   **视觉一致性锁**：
    *   球衣颜色为纯红色，背部为金色 7 号，无英文字母姓名，胸前为金色飞鸟。
    *   逆光夕阳打在草地与球员大腿边缘，反射金色 Rim 边。

### VGU 2：暴力抽射与破网绝平 (TIME: 10.0s - 20.0s)
*   **承载节拍**：B04 (抽击)、B05 (子弹时间越墙)、B06前半部分 (皮球飞越急坠入网与比分改写)
*   **镜头内容**：总裁立定左倾爆抽星纹足球，足球受击变形飞上半空，以子弹慢镜擦过 5 名白衣护裆防守队员（白色人墙）的发梢，随后在空中产生急坠弧线，钻入右上角球兜，挂网隆起。**比分板霓虹闪烁，比分改写跳转为 `RED 3 - 3 WHT`**。
*   **动作连续性控制点**：
    *   10.0s 击球前总裁的跑步惯性与 10.1s 支撑脚落地击球的侧倾 30 度动作无缝衔接；
    *   足球起飞时的位置坐标 $(0,0,-1.5)$ 必须完全继承自 VGU 1 的结束原点；
    *   足球越墙轨迹 $[Z=7.65]$ 需和 B06 破网急坠轨迹 $[Z=16.5]$ 保持曲线动势平滑过度。
*   **视觉一致性锁**：
    *   白衣防守队员身穿纯白色无字球衣，背部非 7 号蓝色数字，在半空中呈现搞笑的皮克斯惊恐表情。
    *   大屏比分板为霓虹灯管外框，比分更新在一阵红金灯光爆裂与像素焰花中切换完成。

### VGU 3：多机位史诗慢镜回放 (TIME: 20.0s - 30.0s)
*   **承载节拍**：B06后半部分 (扑空落地惯性)、B07 (多机位慢动作回放)
*   **镜头内容**：鸭哥门将极限鱼跃扑空后的落地摔滚倒地；切入两个逆光高清慢镜回放机位：① 网底逆光视角展示足球擦过鸭哥橙色乳胶大手套，高速自转砸入网角；② 逆光低机位全景展示皮球飞越白衣人墙、急下坠入网的香蕉轨迹。
*   **动作连续性控制点**：
    *   鸭哥鱼跃扑空至落地滚动的物理惯性在 VGU 3 开场需要完美承接；
    *   两段慢动作回放中的足球自转颗粒、网底凹陷振幅需在视觉上与 VGU 2 的入网瞬间相匹配。
*   **视觉一致性锁**：
    *   鸭哥戴着显眼、厚重、超大比例的亮橙色手套。
    *   日落金边强逆光，画面带有梦幻发光的半透明金色光斑和水汽粒子。

### VGU 4：王者 SIUUU 庆祝定格 (TIME: 30.0s - 40.0s)
*   **承载节拍**：B08 (狂奔、跳跃转体与 SIUUU 定格)
*   **镜头内容**：总裁大口吼叫狂奔向右下角角旗区，高高跃起顺时针转体 180 度。双脚着地马步定格摆开双手，脚下爆开一圈耀眼的半透明红金魔法激波和飞扬草屑。总裁背部 7 号正对镜头歪嘴自信笑定格，队友飞奔滑跪涌来，鸭哥在背景中抱头捶地沮丧，比分板 HUD 锁定在 3-3。
*   **动作连续性控制点**：
    *   总裁狂奔起跳的起点坐标与空中腾空最高点的转体 180 度动能平滑过度；
    *   红衣队友从左侧滑跪冲入角旗区的滑行阻尼感与草屑溅起方向相符。
*   **视觉一致性锁**：
    *   总裁背部亮金 7 号强锁死面向镜头；
    *   **落地激波**：半透明红色和金色交织的环形扩散波纹，空中飘着可爱的发光星星与卡通金币粒子。
    *   比分看板背景恒定显示 `RED 3 - 3 WHT`，下方 `TIME` 锁定在 88:00。

## 2. VGU 生成连续性保障策略

1.  **首尾帧色彩与灯光继承**：在生成下一个 VGU 时，必须引用前一个 VGU 的最后一帧（Last Frame Image）作为图像引导（Image Prompt / Image-to-Video Seed），确保草地的亮绿饱和度、黄昏落日天空的红金渐变和探照灯冷蓝光斑在两段视频间不产生忽明忽暗的闪烁。
2.  **物理形变过渡安全带**：形变特效（如足球击打扁平化、鸭哥扑救长臂拉伸、SIUUU魔法波）只能在单元中段发生，视频生成单元的拼接点（10.0s、20.0s、30.0s）角色和道具必须恢复常态比例，防止由于形变数据在拼接处被视频模型误解导致鬼畜。
3.  **轴线锁定监察**：Z 轴进攻方向与 X 轴左右防守方向在每个 VGU 的 Prompt 编写时必须强制重复声明，防止视频模型自主产生摄像机镜头的左右翻转越轴。
